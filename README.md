##  The Problem

India's **7 million+ food delivery workers** on Zomato and Swiggy are the backbone of the digital economy. But every time an external disruption hits — heavy rain, a curfew, extreme pollution, a platform outage — they lose **20–30% of their monthly earnings** with zero protection.

> *"Currently, gig workers have no income protection against these uncontrollable events. When disruptions occur, they bear the full financial loss with no safety net."* — DEVTrails Problem Statement

<div align="center">

| Disruption Type | Examples | Impact |
|---|---|---|
|  **Environmental** | Extreme heat, Heavy rain, Floods, Severe pollution | Cannot work outdoors / Deliveries halted |
|  **Social** | Unplanned curfews, local strikes, sudden market/zone closures | Inability to access pickup/drop locations |
|  **Platform** | App crashes, server outages, GPS failures | Orders cannot be accepted or completed |

</div>

| Metric | Value |
|---|---|
| Food delivery workers (Zomato + Swiggy) | ~7 million |
| Workers with any income protection | ~9% |

| Total sector income loss annually | ~₹900 Cr+ |

##  Our Solution

**EliteSuraksha** is a **parametric income insurance platform**. Instead of asking workers to prove what they lost, we pre-define objective trigger conditions. When a trigger fires, the payout happens automatically — no forms, no waiting.

```
╔══════════════════════════════════════════════════════════════╗
║  TRADITIONAL INSURANCE                                       ║
║  Event → File Claim → Investigation → Assessment → Payout   ║
║  Timeline: 7–30 days                                         ║
╠══════════════════════════════════════════════════════════════╣
║  ELITESURAKSHA (PARAMETRIC)                                  ║
║  Event → AI Detects → Validates → Fraud Check → Payout      ║
║  Timeline: < 2 minutes                                       ║
╚══════════════════════════════════════════════════════════════╝
```

**Three core principles:**
1. **Automate everything** — zero human touchpoints in the claim pipeline
2. **Use objective data** — payouts based on real sensor/API data, not self-reported loss
3. **Weekly pricing** — matches how gig workers actually earn and budget

##  How It Works

```
1. WORKER ONBOARDS
   Phone OTP → Aadhaar eKYC → Employment Verification (Swiggy/Zomato screenshot)
   → Location captured → Risk score calculated → Weekly premium set → AutoPay activated

2. REAL-TIME MONITORING (always running, every 5 minutes)
   Weather APIs + AQI APIs + Traffic APIs + News APIs + Platform HTTP probes
   → Risk score updated → Alert triggered if threshold approaching

3. PARAMETRIC TRIGGER FIRES
   Condition met (e.g., rain > 50mm/hr)
   → Multi-source confirmation (2+ independent APIs must agree)
   → Worker GPS validated in affected zone
   → Worker activity validated (active within last 10 minutes)

4. FRAUD CHECK (parallel processing, < 5 seconds)
   GPS spoof detection + Duplicate claim check + Behavioral AI + Device fingerprint

5. INSTANT PAYOUT
   Claim approved → UPI Instant triggered
   → Worker receives push notification: "₹600 credited to your UPI" in < 2 minutes
```

##  Weekly Premium Model

> **Why Weekly?** As required by DEVTrails — gig workers earn week-to-week. A weekly premium cycle matches their cash flow and builds habit without large upfront commitment.

```
Weekly Premium = Base (₹20) × (1 + Risk Multiplier) × Loyalty Discount × Tier Factor

Risk Multiplier = Weather(0.40) + AQI(0.25) + Traffic(0.20) + Social(0.10) + Platform(0.05)
```

| Component | Range | Notes |
|---|---|---|
| Base Premium | ₹20/week | Baseline for all workers |
| Weather Risk | 0.0 – 0.50 | Peaks during monsoon (Jun–Sep) |
| Pollution Risk | 0.0 – 0.30 | Higher in North India winter |
| Traffic Risk | 0.0 – 0.30 | Based on worker's delivery zone |
| Social Risk | 0.0 – 0.25 | Elections, known unrest areas |
| Platform Risk | 0.0 – 0.20 | Historical outage frequency |
| Loyalty Discount | 0% – 15% | 3% off per continuous quarter |
| Standard Tier (1.4×) | Max ₹600/event | Default tier |
| Premium Tier (1.9×) | Max ₹1,000/event | Higher coverage |

**Worked Example:**
```
Low Risk (Pune, Summer):    ₹20 × (1 + 0.11) × 0.94 × 1.4 = ₹29/week
High Risk (Mumbai, Monsoon): ₹20 × (1 + 0.42) × 1.00 × 1.4 = ₹40/week
```

> **AI Enhancement (Phase 2):** ML model will charge ₹2–3 less per week for workers operating in zones historically safe from waterlogging — hyper-local pricing as suggested in the DEVTrails tips.

##  Parametric Triggers

All triggers are based on **objective, third-party measurable data** — not self-reported loss. Coverage is for **income loss only** — no vehicle, health, or accident payouts.

| # | Trigger | Threshold | Source | Payout (Standard) |
|---|---|---|---|---|
| 1 |  Heavy Rain | Rainfall > 50mm/hr | OpenWeatherMap + IMD | ₹300 – ₹600 |
| 2 |  Flood Warning | NDMA Orange/Red alert | NDMA official API | ₹500 – ₹900 |
| 3 |  Hazardous AQI | AQI Index > 300 | CPCB + WAQI | ₹200 – ₹500 |
| 4 |  Extreme Heat | Temperature > 42°C | OpenWeatherMap | ₹200 – ₹400 |
| 5 |  Curfew / Bandh | Official order detected | NewsAPI + Govt portal | ₹300 – ₹600 |
| 6 |  Platform Outage | Downtime > 2 hours | HTTP synthetic probes | ₹200 – ₹400 |

**Multi-Signal Confirmation — prevents false triggers:**

| Trigger | Required Confirmation |
|---|---|
| Rain | Weather API + worker GPS in affected zone |
| Curfew | 2 of 3 sources: News + Govt portal + Twitter geo |
| Platform Outage | All 3 HTTP probes from different regions fail |
| Flood | At least 2 official sources (NDMA + IMD) confirm |

##  AI/ML Integration

Per DEVTrails technical requirements — AI/ML is integrated across three functions:

### 1. Dynamic Risk Assessment & Premium Calculation
- **Model:** Gradient Boosted Trees (XGBoost)
- **Input:** 47 real-time variables — weather, AQI, traffic, social signals, time-of-day, historical patterns
- **Output:** Continuous risk score 0.0–1.0 → directly drives weekly premium
- **Key feature (Phase 2):** Hyper-local pricing at 500m × 500m grid — worker in flood-prone zone pays ₹3 more than worker 2km away on elevated ground

### 2. Predictive Risk Forecasting (Phase 2)
- **Model:** LSTM (Long Short-Term Memory) neural network
- **Input:** 72-hour rolling time-series of environmental data
- **Output:** Probability of trigger breach in next 1hr / 3hr / 6hr
- **Use case:** Pre-emptive coverage escalation; system can warn workers before triggers fire

### 3. Intelligent Fraud Detection (Phase 3)
- **GPS Spoofing:** Device attestation API + impossible-speed detection between GPS pings
- **Behavioral Anomaly:** Isolation Forest model on claim patterns, timing, and frequency
- **Peer Comparison:** If worker claims but >80% of nearby peers did not → flagged
- **Historical Validation:** Cross-check claimed weather event against historical sensor data
- **Duplicate Prevention:** Unique event IDs enforced across all accounts

##  Getting Started

### Prerequisites
- **Node.js 18+** → [nodejs.org](https://nodejs.org/en/download)
- **VS Code** → [code.visualstudio.com](https://code.visualstudio.com/)

### Installation
```bash
# Clone
git clone https://github.com/YOUR_USERNAME/elite-suraksha.git
cd elite-suraksha

# Install
npm install

# Run
npm run dev
# → Opens at http://localhost:3000
```

> **Windows EPERM error fix:**
> ```
> npm config set prefix "%APPDATA%\npm"
> npm install
> ```
> If still failing, copy to C: drive: `xcopy D:\elite-suraksha C:\elite-suraksha /E /I /H`

### Optional: Live Weather Data
```bash
cp .env.example .env
# Add free key from https://openweathermap.org/api
# VITE_WEATHER_API_KEY=your_key_here
npm run dev
# Dashboard shows 🟢 Live API when active
```
